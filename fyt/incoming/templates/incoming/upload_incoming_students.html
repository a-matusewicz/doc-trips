The database expects to find the following columns in the CSV file:

  • Id (this is the students NetId)
  • Formatted Fml Name
  • Class Year
  • Gender
  • Birthday
  • Fine Ethnic Code
  • EMail
  • Blitz
  • PR Phone
  • PR Street 1
  • PR Street 2
  • PR City
  • PR State
  • PR Zip
  • Pr Nation Name
Yes, 'Pr Nation Name' has a lowercase 'r' for whatever reason. These column names were pulled from last year's incoming student data. Let me know if the headers have changed dramatically and I can tweak the expectations of the input. The order of the columns in the CSV file does not matter. Any extra columns will be ignored.

If an incoming student already exists in the database for this trips year (determined by comparing NetIds) the database skips importing that information.
